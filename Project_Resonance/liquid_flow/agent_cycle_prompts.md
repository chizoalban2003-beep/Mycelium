# Agent Cycle Prompts (Perturbation -> Survival -> Sedimentation)

Use these prompts in Cursor Agent Mode for repeatable evolution loops.

## Step A: Perturbation (Injection)

Prompt:

"Inject an Energy Spike into the current physics engine and autonomy policy.
Introduce a new `dissonance` variable representing <YOUR_FORCE_NAME>.
Show which current constants or heuristics are most affected.
Return:
1) impacted modules,
2) temporary (gaseous) experiments,
3) safe rollback plan."

## Step B: Survival Cycle (Metabolism)

Prompt:

"Run a Survival Check for active modules.
Identify high-friction code paths (high complexity, low impact, low usage).
Recommend which liquid modules should be:
- stabilized,
- recycled (deleted),
- split into clearer interfaces.
Return a friction-ranked table."

## Step C: Sedimentation (Hardening)

Prompt:

"Identify logic patterns that survived the last 3 spikes.
Crystallize them into `Project_Resonance/bedrock/bedrock_manifest.json`.
For each pattern, include:
- confidence,
- supporting evidence references,
- cooling threshold,
- dissolve threshold,
- owner."

## Observer Prompt (Before coding)

"@Codebase Generate a `State of the Fluid` report:
- Where is dissonance highest?
- Where is bedrock too rigid?
- Which liquid modules are blocked by unclear interfaces?
- Which gaseous experiments deserve one more cycle?"
