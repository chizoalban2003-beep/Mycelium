# Entropy Logs

This folder keeps the ecosystem's historical stress record:

- energy spikes and what changed
- survival checks and modules recycled
- failures, regressions, and recoveries
- crystallization events and hardening outcomes

Use one markdown file per major cycle:

- `spike_0001.md`
- `spike_0002.md`
- ...

Each log should include:

- spike cause
- affected modules
- fitness/failure metrics
- what was dissolved
- what was hardened
- residual risks
