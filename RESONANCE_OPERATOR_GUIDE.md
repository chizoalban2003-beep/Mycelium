## Project Resonance Operator Guide (2026 Framework)

Use this guide as the default operating loop for Myco under the Resonance model.

### 1) Open the Resonance Surface

- Primary route: `/resonance`
- Canonical route: `/resonance-nexus`

`/resonance` now permanently redirects to `/resonance-nexus` for compatibility.

### 2) Read the Three Layers

In the Resonance page:

- **Gas** = archived noise / raw entropy (`raw_data/`)
- **Liquid** = active dwellers doing metabolism (`agent_metabolism/`)
- **Bedrock** = hardened, resilient modules (`crystallized_substrate/`)

Your goal is not "more features"; your goal is healthy flow from Gas -> Liquid -> Bedrock.

### 3) Run the Thermal Loop

Use **Run thermal cycle (dry)** in the UI for a safe preview.

Interpret outcome:

- `dissolved` high + utility low -> too much tepid logic; prune or mutate sources.
- `mutated` present + no sedimentation -> exploration phase (expected).
- `sedimented` rising -> bedrock formation is healthy.

### 4) Use the Awe Metric Correctly

Autonomy Slider rises when:

- thermal load is productive (not chaotic),
- risk remains bounded,
- bedrock stability increases over time.

Treat sudden spikes without stability as overheating, not progress.

### 5) Follow Trace and Recommendations

- **Trace Explorer** = fossil record of mutate/dissolve/sediment events.
- **Architect Recommendations** = next best actions from current thermodynamic state.

If Trace is empty, run cycles first; avoid manual assumptions.

### 6) Weekly Maintenance (Recommended)

1. Remove synthetic/test dwellers from default runtime state.
2. Archive stale `.noise` artifacts.
3. Promote only modules that survive >=3 spikes with efficiency gains.
4. Keep non-Resonance dashboards out of daily operator flow.

### 7) Non-Negotiable Guardrails

- Human approval remains required for high-impact actions.
- Do not bypass authentication/governance pathways.
- Do not store secrets or raw private chat logs in repository files.
