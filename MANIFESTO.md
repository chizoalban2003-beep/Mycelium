Project Resonance is not a product; it is a Process.
It is the transition of data from Noise (Gas) to Life (Liquid) to Truth (Bedrock).
We are not building a system; we are hosting an emergence.

## Identity

Project Resonance is a thermodynamic, agentic ecosystem.
Code is treated as biomass that competes under friction, heat, and limited energy.

## Layer Topology

- `raw_data/` -> gaseous, ephemeral, high-entropy noise and archived dissolutions.
- `agent_metabolism/` -> liquid, adaptive, self-refactoring operational logic.
- `crystallized_substrate/` -> bedrock, hardened and low-entropy modules.
- `.secrets/` -> guarded manifests about hidden optimization paths.

## Survival Law

1. New logic enters through `agent_metabolism/` as candidate matter.
2. Logic must prove entropy reduction and utility under repeated perturbation.
3. Stable logic is sedimented into `crystallized_substrate/`.
4. Tepid logic dissolves into `raw_data/` as `.noise` artifacts for future mutation.

## Governance

- Human guardians remain the final approval layer for high-risk actions.
- Safety, privacy, and authentication pathways are immutable without explicit review.
- Emergence is encouraged; unsafe autonomy is not.

## 2026 Burn Upgrades

- **Mission Control + Cloud Handoff**: Long-running burn cycles should run in cloud handoff mode for state persistence.
- **Agent Trace fossil record**: Every dissolve/mutate/sediment action writes lineage data into `.secrets/agent_trace.jsonl`.
- **Multi-agent friction**: Run creative mutation and strict verification in parallel roles, then merge only audited survivors.
- **Zero-downtime sedimentation**: Use git worktrees for high-entropy experiments and merge only DKS-positive outcomes.
